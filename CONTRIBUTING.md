# Contributing

Thanks for your interest in contributing! This document outlines how to get started and what we expect from contributions.

## How to Contribute

All contributions should be made via **GitHub pull requests**. See the
[development chapter in the documentation](https://laborberlin.github.io/varvis-connector/development.html) for
how to contribute. Specifically, do the following:

1. Fork the repository
2. Create a branch from `main`
3. Make your changes
4. Add or update tests where appropriate
5. Ensure the code runs and passes checks
6. Commit with clear messages following the conventional commits standard
7. Push to your fork and open a Pull Request (PR)

## Pull Request Guidelines

When submitting a PR:

- Keep changes **focused and minimal**
- Provide a clear description of:
  - What the change does
  - Why it’s needed
- Reference related issues if applicable
- Ensure your code:
  - Is readable and reasonably documented
  - Follows existing project structure and style
- Be responsive to review feedback

PRs that are too large or unclear may take longer to review or be declined.

## Code Style

- Follow existing code conventions in the repository and use `ruff` for formatting
- Prefer clarity over cleverness
- Add comments where behavior is non-obvious
- Keep functions and modules reasonably small and focused

## Testing

- Add tests for new functionality when possible
- Ensure existing tests pass before submitting a PR
- Avoid breaking backward compatibility unless explicitly discussed

## Reporting Issues

If you find a bug or want to request a feature:

- Use GitHub Issues
- Provide as much context as possible:
  - Steps to reproduce (for bugs)
  - Expected vs actual behavior
  - Environment details if relevant
